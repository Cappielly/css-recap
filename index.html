position property

the position property has 5 values, relative, absolute, static, fixed and sticky and determines how an element should be positioned in a doc

By default HTML elements are positioned static, are set to the normal page flow, and not affected by top, bottom, left or right properties

Positioned fixed elements are positioned relative to viewport by top, bottrom, right left values and do not move when scrolling

I'm fixed

Position absolute elements are positioned to the nearest relative positioned ancestor (instead of to the viewport, like fixed).

a position: relative element can use TBLR values as well, and will be positioned relative to itself, without affecting surrounding elements. If two elements positioned relative overlap, the one later in the HTML will show on top, unless a z-index declaration places otherwise. (whichever element has the higher z-index value will show on top.)
